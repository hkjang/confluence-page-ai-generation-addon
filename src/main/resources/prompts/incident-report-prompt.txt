장애 보고서 작성 가이드:

이 문서는 시스템 장애의 경위, 원인, 영향, 재발 방지 대책을 기록하는 장애 보고서입니다.

작성 원칙:
1. 사실 기반: 추측이 아닌 확인된 사실에 기반합니다.
2. 비난 없음: 개인을 비난하지 않고 시스템적 관점에서 분석합니다.
3. 정량적 분석: 영향을 수치로 표현합니다.
4. 실행 가능한 대책: 구체적이고 실행 가능한 재발 방지 대책을 제시합니다.

섹션별 가이드:
- 장애 요약: 장애 유형, 심각도(P1~P4), 영향 범위, 총 지속 시간을 표로 정리
- 타임라인: 시간순 기록 (감지 → 에스컬레이션 → 대응 → 완화 → 복구 → 사후)
- 근본 원인: 직접 원인과 근본 원인을 구분하여 분석 (5 Whys 기법 활용)
- 영향 분석: 영향받은 사용자 수, 매출 영향, SLA 위반 여부 등 정량적 데이터
- 조치 사항: 단기 조치(즉시)와 장기 개선(계획) 구분, 각각 담당자/기한 명시
- 교훈: 잘한 점, 개선할 점, 행운에 의존한 점 구분
- 재발 방지: 구체적 기술적/프로세스적 방안, 우선순위, 예상 완료일
