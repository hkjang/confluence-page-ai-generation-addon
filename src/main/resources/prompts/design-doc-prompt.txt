설계서 작성 가이드:

이 문서는 시스템 또는 기능의 기술 설계를 상세히 기술하는 설계서입니다.

작성 원칙:
1. 기술적 정확성: 구현 가능한 수준의 구체적인 설계를 제시합니다.
2. 의사결정 근거: 설계 선택의 이유와 트레이드오프를 명확히 합니다.
3. 다이어그램 활용: 아키텍처, 시퀀스, 데이터 흐름을 시각적으로 설명합니다.

섹션별 가이드:
- 개요: 프로젝트/기능의 배경, 목적, 범위 (2-3문단)
- 요구사항: 기능적 요구사항(FR)과 비기능적 요구사항(NFR)을 분리하여 ID 부여
- 아키텍처: 전체 시스템 구조, 컴포넌트 간 관계, 데이터 흐름
- 상세 설계: 클래스/인터페이스 설계, API 명세, 데이터 모델
- 구현 계획: 단계별 구현 일정, 마일스톤, 의존성
- 대안 분석: 검토한 대안, 각 장단점, 최종 선택 이유
- 리스크: 기술적 리스크와 완화 전략 (표 형식)
